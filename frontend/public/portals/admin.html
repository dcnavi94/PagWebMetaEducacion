                Flujo de sincronización
                1. Materias: se crea o vincula un curso mediante un shortname estable.
                2. Usuarios: se buscan por username/correo y se crean si no existen.
                3. Roles: Moodle recibe el rol configurado para alumno o docente.
                4. Matrículas: las asignaciones locales inscriben al usuario en el curso.
                5. Acceso: el portal abre Moodle usando la URL pública configurada.
                Solución de problemas

                Regenera el token, confirma el servicio externo y revisa que esté habilitado.

                Dentro de Docker usa http://moodle:8080/webservice/rest/server.php. En el navegador usa http://localhost:8081.

                Añade al servicio externo todo permiso marcado como faltante en esta pantalla.
